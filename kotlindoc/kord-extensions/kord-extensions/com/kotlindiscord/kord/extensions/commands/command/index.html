Command

class Command

Class representing a single command.

You shouldn't need to use this class directly - instead, create an Extension and use the command function to register your command, by overriding the Extension.setup function.

Fields

Name Description
lateinit var name: String

The name of this command, for invocation and help commands.
var description: String

A description of what this function and how it's intended to be used.

This is intended to be made use of by help commands.
var enabled: Boolean

Whether this command is enabled and can be invoked.

Disabled commands cannot be invoked, and won't be shown in help commands.

This can be changed at runtime, if commands need to be enabled and disabled dynamically without being reconstructed.
var hidden: Boolean

Whether to hide this command from help command listings.

By default, this is false - so the command will be shown.
var signature: String

The command signature, specifying how the command's arguments should be structured.

You may leave this as it is if your command doesn't take any arguments, you give the signature function a dataclass to generate a signature, or you can specify this in the Extension.command builder function if you'd like to provide something a bit more specific.
var aliases: Array<String>

Alternative names that can be used to invoke your command.

There's no limit on the number of aliases a command may have, but in the event of an alias matching the name of a registered command, the command with the name takes priority.
val extension: Extension

The Extension that registered this command.

check

fun check(vararg checks: suspend (MessageCreateEvent)->Boolean)

Define a check which must pass for the command to be executed.

A command may have multiple checks - all checks must pass for the command to be executed. Checks will be run in the order that they're defined.

This function can be used DSL-style with a given body, or it can be passed one or more predefined functions. See the samples for more information.

call

suspend fun call(event: MessageCreateEvent, args: Array<String>)

Execute this command, given a MessageCreateEvent.

This function takes a MessageCreateEvent (generated when a message is received), and processes it. The command's checks are invoked and, assuming all of the checks passed, the command body is executed.

If an exception is thrown by the command body, it is caught and a traceback is printed.
