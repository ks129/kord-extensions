package com.kotlindiscord.kord.extensions.commands

Classes

Name Description
class ArgumentParser

Class in charge of converting string arguments for commands into fully-typed data classes.

You most likely don't need to touch this yourself - it's part of the commands framework.
class Command

Class representing a single command.

You shouldn't need to use this class directly - instead, create an Extension and use the command function to register your command, by overriding the Extension.setup function.
class CommandContext

Light wrapper class representing the context for a command's action.

This is what this refers to in a command action body. You shouldn't have to instantiate this yourself.
